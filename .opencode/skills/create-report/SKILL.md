# Create Report

Generate a professional report saved as a local file the user can download. Supports two output formats: HTML (rich visual report viewable in the browser) and Word (.docx).

## Example Trigger Prompts

- "Create a report on our migration status"
- "Generate an analysis document for the Q3 review"
- "Write a summary report of these findings"
- "Build a Word doc with our team's recommendations"

## Choosing the Format

- If the user asks for a Word doc, .docx, or "document" -> use python-docx to generate .docx
- If the user asks for HTML, a visual report, or does not specify -> generate an HTML file
- If unsure, ask the user which format they prefer

## Instructions

Follow these steps in order:

1. Read and understand the input (topic, data, notes, or content provided by the user).
2. Plan the report structure. Every report must include:
   - Title and date
   - Executive Summary (2-4 sentences)
   - 3-6 content sections with findings, analysis, or recommendations
   - Conclusion or Next Steps
3. Write a Python script to generate the report file.
4. Execute the script using `python3.11`.
5. Tell the user the file path so they can download it.

## Critical: Python Version

ALWAYS use `python3.11` to run the script, NOT `python3`. The system python3 is too old. Example:

```bash
python3.11 build_report.py
```

## Writing Rules

- Use clear section headings that convey the point, not generic labels.
- Keep paragraphs short (2-4 sentences).
- Use bullet points for lists of findings or recommendations.
- Include specific data, numbers, or evidence when available.
- Write in professional business language. No filler, no jargon.

## Word Document (.docx) Template

Use this pattern when generating a Word document. Save as `build_report.py` and run it.

```python
from docx import Document
from docx.shared import Pt, Inches, RGBColor
from docx.enum.text import WD_ALIGN_PARAGRAPH

doc = Document()

style = doc.styles['Normal']
font = style.font
font.name = 'Calibri'
font.size = Pt(11)

# Title
title = doc.add_heading('REPORT TITLE', level=0)
title.alignment = WD_ALIGN_PARAGRAPH.LEFT

doc.add_paragraph('Prepared by: Team Name | Date: YYYY-MM-DD')
doc.add_paragraph('')

# Executive Summary
doc.add_heading('Executive Summary', level=1)
doc.add_paragraph('Summary paragraph here.')

# Content sections
doc.add_heading('Section Title', level=1)
doc.add_paragraph('Content here.')

# Bullet points
doc.add_heading('Key Findings', level=1)
items = ['Finding one', 'Finding two', 'Finding three']
for item in items:
    doc.add_paragraph(item, style='List Bullet')

# Conclusion
doc.add_heading('Conclusion', level=1)
doc.add_paragraph('Wrap-up and next steps.')

filename = 'report-title.docx'
doc.save(filename)
print(f'Saved: {filename}')
```

## Rules

- Always use blank/minimal styling. Dark theme for HTML, clean formatting for Word.
- File names should be descriptive: "migration-analysis-report.html" not "report.html".
- After saving, tell the user the exact file path and how to download it from Dev Spaces.
- For HTML: the report must be a single self-contained file with inline CSS. No external dependencies.
- For Word: use proper heading levels, bullet styles, and page formatting.
- If python-docx is not installed, run `python3.11 -m pip install python-docx` first.
- ALWAYS use `python3.11` to run the script. NEVER use `python3` or `python`.
